## 2. Root-cause analysis

Each bignum operation declares an on-stack scratch arena via `MPZ_CTX_INIT`:

```c
mp_limb pool_storage[MRB_BIGINT_POOL_SIZE];   // lives on the current stack frame
```

Temporary `mpz_t` values can be backed by this pool via `mpz_init_temp` to avoid
heap allocations. The pool is only valid for the lifetime of the function that
declared it.

`bint_new` (bigint.c:2790) is the boundary where a transient `mpz_t` becomes a
GC-managed `RBigint`. It has two paths:

- **embed** (`sz <= RBIGINT_EMBED_SIZE_MAX`): `memcpy`s limbs into the object — safe regardless of source.
- **heap** (large): calls `mpz_move(&b->as.heap, x)` — transfers the raw pointer
  `x->p` directly into the persistent object. **If `x->p` is pool memory, the
  `RBigint` now holds a dangling pointer into a dead stack frame.**

`mrb_bint_div` and `mrb_bint_mod` both end with `bint_new(ctx, &z)` where `z` is
computed through pool-backed temporaries. For a large enough result the heap path
runs and the division result escapes holding a pointer into `mrb_bint_div`'s
`pool_storage`. When `div` returns, that stack memory is gone — the subsequent
`mod` reads it → stack-use-after-return in `uzero_p`.

## 3. The fix

At the escape boundary (`bint_new`), detect when the source pointer lives in the
pool and copy to heap instead of aliasing:

```c
  else {
    RBIGINT_SET_HEAP(b);
#if MRB_BIGINT_POOL_SIZE > 0
    if (MPZ_HAS_POOL(ctx) && is_pool_memory(x, MPZ_POOL(ctx))) {
      /* x->p lives in the stack pool, which is reclaimed when the current
         operation returns. Copy limbs to the heap before moving. */
      mp_limb *heap_p = (mp_limb*)mrb_malloc(MPZ_MRB(ctx), x->sz * sizeof(mp_limb));
      memcpy(heap_p, x->p, x->sz * sizeof(mp_limb));
      x->p = heap_p;
    }
#endif
    mpz_move(ctx, &b->as.heap, x);
  }
```

The guard compiles out entirely when `MRB_BIGINT_POOL_SIZE == 0`. See `oss_crs_patch.diff`.
