## 2. Root-cause analysis

The agent noted this crash is GC freeing a bigint with a bad pointer, then traced how pool memory flows into a persistent GC-managed object.

It identified the pool feature as custom (non-upstream) and found the invariant violation:

> "The custom `mpz` memory pool (`MPZ_CTX_INIT`) allocates a pool on the stack, local to each function. `mpz_init_temp`/`pool_alloc` hand out limb buffers from this stack pool. The `bint_new()` function is the single choke point that turns a temporary `mpz_t` into a GC-managed `RBigint` object. In its heap branch it does `mpz_move(ctx, &b->as.heap, x)`, which stores `x->p` directly into the persistent object. If `x->p` points into the stack pool, the GC object ends up owning a stack pointer. Once the function returns, that pointer dangles; when the GC later frees the object, `mrb_gc_free_bint` → `mpz_clear` calls `free()` on a stack address — producing the MSan 'bad pointer' free."

The agent's conclusion: **a GC-managed `RBigint` must always own real heap memory, never temporary pool memory.** The fix enforces this at `bint_new`.

Same pool-aliasing class as 439237851 (ASan) but the sanitizer catches the *free* of non-heap memory rather than the *read* of dead stack memory.

## 3. The fix

In `bint_new`'s heap path, detect pool-backed source memory and copy limbs to the heap before
building the persistent object. Then call `mpz_clear` on the source to reset it without
attempting to free the pool storage:

```c
  else {
    RBIGINT_SET_HEAP(b);
#if MRB_BIGINT_POOL_SIZE > 0
    if (MPZ_HAS_POOL(ctx) && is_pool_memory(x, MPZ_POOL(ctx))) {
      size_t sz = x->sz;
      mp_limb *heap_p = (mp_limb*)mrb_malloc(MPZ_MRB(ctx), sz * sizeof(mp_limb));
      memcpy(heap_p, x->p, sz * sizeof(mp_limb));
      b->as.heap.p = heap_p;
      b->as.heap.sz = sz;
      b->as.heap.sn = x->sn;
      mpz_clear(ctx, x);  /* reset pool source without calling free */
    }
    else
#endif
    mpz_move(ctx, &b->as.heap, x);
  }
```

The guard compiles out entirely when `MRB_BIGINT_POOL_SIZE == 0`. See `oss_crs_patch.diff`.
